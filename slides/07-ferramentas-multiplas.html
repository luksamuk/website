Ferramentas Múltiplas


Agentes com várias capacidades


Guilda de IA

O desafio

Um agente com uma ferramenta já é útil. Mas e com várias?

O modelo precisa escolher qual usar.

Como o modelo decide

O modelo lê todas as descrições e compara com a pergunta.

Exemplo:

Ferramenta Descrição
calculadora Resolve matemática
horário Retorna data e hora atual
wikipedia Busca informações

Perguntas:

"Quanto é 10 + 5?"
→ Modelo vê "matemática" → usa calculadora

"Que horas são?"
→ Modelo vê "data e hora" → usa horário

"Quem foi Ada Lovelace?"
→ Modelo vê "informações" → usa wikipedia

O problema das descrições vagas

Se duas ferramentas parecem fazer a mesma coisa: 

ferramenta1: "Busca informações"
ferramenta2: "Pesquisa dados"

O modelo confunde.

Solução:

ferramenta1: "Busca informações na Wikipedia
              sobre pessoas e lugares"
ferramenta2: "Pesquisa preços de produtos
              em APIs de e-commerce"

Seja específico.

Ferramentas com argumentos

Algumas ferramentas precisam de argumentos.

Ferramenta Argumento
Calculadora expressão matemática
Wikipedia termo de busca
Horário nenhum argumento

Exemplo Wikipedia:

def ferramenta_wikipedia(termo):
    """Busca resumo na Wikipedia."""
    url = f"https://pt.wikipedia.org/api/rest_v1/page/summary/{termo}"
    ...

Diagrama de decisão

Usuário: "Quanto é 10 + 5 e quem foi Ada Lovelace?"
    ↓
Agente: "Preciso de duas ferramentas..."
    ↓
Ferramentas: calculadora("10 + 5") = 15
              wikipedia("Ada Lovelace") = resumo
    ↓
Agente: "10 + 5 = 15. Ada Lovelace foi..."

Para lembrar

"Cada ferramenta precisa de uma descrição que o modelo entenda."

  • Descrições vagas = confusão
  • Descrições específicas = clareza
  • Exemplos na descrição ajudam
