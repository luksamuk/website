Dando poderes ao agente

O problema do LLM puro

LLMs são ruins em matemática. 

Pergunta: "Quanto é 234 × 987?"

O modelo não calcula. Ele prevê a resposta baseado em padrões. Pode acertar, pode errar.

A solução: ferramentas

Ferramenta = função Python que o agente pode chamar.

Como funciona:

  1. Modelo recebe descrição da ferramenta
  2. Analisa a pergunta
  3. Decide: preciso dessa ferramenta?
  4. Se sim, gera chamada no formato correto
  5. Python executa e devolve resultado
  6. Modelo usa resultado na resposta

Exemplo: calculadora

def calcular(expressao):
    """Resolve expressão matemática."""
    return str(eval(expressao))

# Exemplo:
calcular("234 * 987")  # "230958"

O segredo: descrição clara

O modelo decide baseado na descrição.

Descrição ruim:

"Calculadora."

→ Modelo não sabe quando usar.

Descrição boa:

"Resolve expressões matemáticas. Use quando precisar fazer cálculos. Ex: '2 + 2', '10 * 5', '(5 + 3) / 2'."

→ Modelo entende quando e como usar.

O formato de chamada

O modelo precisa gerar algo que Python entenda. 

USAR_FERRAMENTA: calculadora
ARGUMENTO: 234 * 987

Seu código:

  1. Detecta isso na resposta
  2. Executa a função
  3. Devolve o resultado para o modelo

Diagrama

Usuário: "Quanto é 234 × 987?"
    ↓
Agente: "Preciso da calculadora"
    ↓
Ferramenta: calcular("234 * 987") = 230958
    ↓
Agente: "230.958"

Para lembrar

"Ferramenta transforma conversa em ação."

LLM puro: só fala Agente com ferramenta: fala E faz
