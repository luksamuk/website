RAG


O agente que lê documentos


Guilda de IA

O problema

LLMs não conhecem:

  • Seus documentos pessoais
  • PDFs da empresa
  • Notas privadas
  • Informações recentes (pós-treinamento)

A solução: RAG

RAG = Retrieval Augmented Generation

Fluxo:

  1. Chunking → divide documentos em pedaços
  2. Embeddings → transforma texto em números
  3. Vector Store → guarda os vetores
  4. Retrieval → busca pedaços relevantes
  5. Generation → LLM responde com contexto

Analogia

  • Sem RAG: aluno fazendo prova de memória
  • Com RAG: aluno fazendo prova com livro aberto

Chunking

Dividir texto em pedaços menores. 

def dividir_em_chunks(texto, tamanho=200):
    chunks = []
    for i in range(0, len(texto), tamanho):
        chunks.append(texto[i:i+tamanho])
    return chunks

Por quê? Documentos grandes são difíceis de buscar. Pedaços pequenos = busca mais precisa.

Embeddings (simplificado)

Texto → números = vetor

Vetores parecidos = textos parecidos.

Busca semântica: encontra significado, não só palavras.

Matryoshka Embeddings

Embeddings que podem ser truncados.

  • Normal: 768 dimensões fixas
  • Matryoshka: 768, 512, 256, 128… (flexível)

Vantagem: economiza espaço e tempo.

Fluxo RAG completo

Documento → Chunks → Embeddings → Vector Store
                                         ↓
Usuário pergunta                        ↓
      ↓                                  ↓
Pergunta → Embedding → Busca similar → Chunks relevantes
                                              ↓
                              LLM recebe pergunta + chunks
                                              ↓
                                         Resposta

Para lembrar

"RAG = buscar antes de responder."

  • Chunking: divide em pedaços
  • Embeddings: transforma em números
  • Retrieval: encontra o relevante
  • Generation: LLM responde
